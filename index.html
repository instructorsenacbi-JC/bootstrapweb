DM-XR

Dispositivo interactivo para simulación y validación en Diseño Industrial mediante Realidad Extendida, orientado a reducir tiempos de iteración, costos de prototipado y niveles de incertidumbre antes de la fabricación.

Validación inmersiva de productos

Menos riesgo. Más precisión.

El sistema permite probar, modificar y analizar productos en escenarios físicos y virtuales, fortaleciendo la toma de decisiones en etapas clave del diseño.

El problema

En los procesos tradicionales de diseño, la validación suele depender del prototipado físico, lo que incrementa costos, tiempos de iteración y consumo de materiales.

Además, muchas tecnologías disponibles son cerradas, poco personalizables y difíciles de adaptar a necesidades pedagógicas o de investigación, especialmente en contextos académicos.

DM-XR surge como una respuesta a esa necesidad: una alternativa propia, flexible y escalable para validar productos mediante experiencias físicas y virtuales.

La solución: DM-XR

DM-XR es un dispositivo interactivo que integra tecnologías de Realidad Extendida como AR, VR y MR con una estructura física y electrónica propia.

Su propósito es permitir la simulación, exploración y validación de productos en tiempo real, articulando entorno físico, visualización 3D, interacción gestual y experimentación inmersiva.

El proyecto combina diseño modular, software en Unity 3D, sensores, control electrónico y recursos XR para fortalecer tanto el proceso de diseño como la enseñanza.

Tecnologías integradas

El proyecto combina plataformas de desarrollo, sensores, electrónica y tecnologías XR para construir una experiencia híbrida de simulación y validación.

Realidad Aumentada

Permite superponer objetos virtuales sobre el entorno real para apoyar procesos de validación.

En el flujo AR se emplea una escena configurada en Unity 3D, una cámara web y un target para sobreponer el modelo virtual y manipularlo desde el sistema.
Realidad Virtual

Genera un entorno inmersivo para explorar productos tridimensionales y controlar su interacción.

El modo VR permite al usuario visualizar e interactuar con el modelo 3D en un espacio inmersivo, articulando visor, sensores y el DM-XR como medio físico.
Unity 3D

Plataforma principal para construir entornos virtuales, interacción y simulación del sistema.

En Unity 3D se importan los modelos, se ajusta la escena y se activan los entornos correspondientes para AR o VR según la necesidad del usuario.
Electrónica y placas

El sistema incorpora una base electrónica con placas, sensores, conectores y lógica de control.

En fases iniciales se emplearon Arduino UNO y Nano, además de una evolución hacia una arquitectura electrónica más personalizada y escalable.
Sensores de movimiento

Capturan orientación, movimientos y gestos para enriquecer la interacción con los objetos virtuales.

Se integran elementos como Leap Motion y sensores microelectromecánicos para detectar movimientos y aportar datos a la simulación XR.
Librerías XR y programación

El proyecto integra C#, C/C++ y librerías especializadas para enlazar hardware y simulación.

Se emplean plugins como SteamVR, Vuforia, Ultraleap y Oculus SDK para habilitar compatibilidad con distintos escenarios de interacción.

Proceso de desarrollo

El proyecto fue desarrollado por fases, desde la revisión técnica inicial hasta la validación funcional del sistema en escenarios educativos.

Se evaluaron necesidades funcionales, componentes clave, hardware compatible, plataformas de desarrollo y criterios de integración tecnológica.

Se generaron bocetos, esquemas y estructuras modulares del sistema, además de la primera organización del hardware y las carcasas.

Se realizó selección, ensamblaje y conexión de componentes electrónicos, sensores, cables, conectores y medios físicos del sistema.

En Unity 3D y Visual Studio se construyeron escenas, scripts, integración de sensores y flujos de simulación XR para interacción en tiempo real.

Se efectuaron pruebas técnicas, funcionales y de usabilidad con usuarios, permitiendo ajustes y consolidación del sistema.

Modo de uso

El sistema puede operar en diferentes escenarios de simulación, especialmente mediante flujos de trabajo en AR y VR.

Flujo básico en Realidad Aumentada

  1. Iniciar la aplicación en Unity 3D.
  2. Importar y ajustar el modelo 3D dentro del escenario.
  3. Seleccionar la escena configurada para AR.
  4. Preparar target impreso, cámara web y conexiones del DM-XR.
  5. Ejecutar la simulación y validar el producto sobre el entorno real.

Flujo básico en Realidad Virtual

  1. Iniciar el proyecto en Unity 3D.
  2. Importar el modelo y configurar escena VR.
  3. Preparar visor, sensores y espacio de desplazamiento.
  4. Conectar el DM-XR al computador.
  5. Ejecutar el entorno inmersivo y manipular el objeto virtual.
